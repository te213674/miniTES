# Инструкция по настройке GitHub Secrets для системы заявок

## Что было реализовано

1. **Формы на сайте** отправляют данные напрямую в GitHub Issues API
2. **GitHub Actions** автоматически отправляет email уведомления при создании Issue
3. **Email уведомления** приходят на `info@minites.ru` через SMTP Яндекса

## Структура файлов

```
.github/workflows/
  └── email-notification.yml    # GitHub Actions workflow
static-site/
  ├── js/

  └── contact.html              # Страница с контактной формой
```

## Как это работает

1. **Клиент заполняет форму** на сайте
2. **JavaScript** отправляет данные на GitHub API с использованием токена
3. **Создаётся Issue** в репозитории с метками и структурированными данными
4. **GitHub Actions** срабатывает на событие `issues: opened`
5. **Actions отправляет email** на `info@minites.ru` через SMTP Яндекса

## Важные замечания

- GitHub токен уже встроен в `github-forms.js`
- Токен имеет доступ только к вашему репозиторию
- Все данные форм сохраняются в Issues, что удобно для отслеживания
- Email уведомления приходят мгновенно после создания Issue

## Troubleshooting

Если заявки не приходят:

1. Проверьте, что секреты правильно добавлены в репозиторий
2. Проверьте логи GitHub Actions: **Actions** → **Send Email Notification on New Issue**
3. Убедитесь, что CORS разрешает запросы к GitHub API (для локальной разработки может потребоваться прокси)

## Контакты для поддержки

Если возникнут проблемы, проверьте:
- Валидность GitHub токена
- Правильность пароля приложения Яндекса
- Настройки безопасности Яндекса (должен быть разрешён доступ по SMTP)
