## Важные примечания

### Токен GitHub API
- **Токен:** [УДАЛЕНО ИЗ БЕЗОПАСНОСТИ]
- **Хранение:** `static-site/js/config.js` (в `.gitignore`)
- **Права:** `repo` (доступ к Issues)
- **Важно:** Не коммитить `config.js` с токеном в репозиторий!

### ⚠️ КРИТИЧЕСКИЕ УРОКИ (18.04.2026)

#### Ошибка с GitHub Token (БОЛЬШЕ НЕ ПОВТОРЯТЬ!)
1. **НИКОГДА не класть PAT токен (ghp_...) в файлы репозитория** — даже в .md файлы с инструкциями
2. **GitHub Secret Scanning автоматически обнаруживает токены** в истории коммитов и БЛОКИРУЕТ push
3. **Токен для клиентского JS хранить ТОЛЬКО в файле, который в .gitignore** (config.js)
4. **Если токен попал в историю:**
   - GitHub даёт ссылку для разблокировки: `https://github.com/owner/repo/security/secret-scanning/unblock-secret/...`
   - Выбрать "I'll fix it later" (токен уже отозван, новый в config.js)
   - НЕ пытаться использовать `git filter-branch` — это сложно и требует approval
5. **GitHub Secrets (YANDEX_EMAIL, YANDEX_PASSWORD) — это ДРУГОЕ:**
   - Они хранятся НА СЕРВЕРЕ GitHub (Settings → Secrets)
   - Они НЕ видны в коде и НЕ попадают в историю
   - Их НЕ НУЖНО удалять/заменять

#### Разница между типами секретов:
| Тип | Где хранится | Видно в коде? | Проблема? |
|-----|-------------|---------------|-----------|
| GitHub Secrets | На сервере GitHub | Нет | Нет |
| PAT Token (ghp_) | В файле config.js (в .gitignore) | Нет (локально) | Нет, если не в репозитории |
| PAT Token (ghp_) | В файле репозитория | Да | ДА — GitHub заблокирует push |
